# 《剖析与实现 MicroClaw》封面设计 Prompt

## 目标

生成一张可直接用于技术书 PDF 封面的竖版封面图。整体气质应当专业、克制、可信、具备书架辨识度，并且明显属于“现代工程类技术书”，而不是科幻海报或产品宣传页。

## 主 Prompt

请设计一本技术书封面，主题是“从零构建一个多渠道 Agent Runtime”。风格可受经典技术出版物启发，但必须保持原创，不得复刻任何现有出版社、品牌、logo、版式或受版权保护的具体视觉元素。

书名：
《剖析与实现 MicroClaw》

副标题：
基于 v0.1.16 源码的多渠道 Agent Runtime 架构、最小实现与生产实践

作者：
不显示作者名

封面定位：
- 这是一本面向工程师的技术书，不是大众科普，不是产品白皮书。
- 视觉上要传达“执行、状态、工具、记忆、治理”这些关键词。
- 封面应当让人联想到“系统性构建”“工程秩序”“长期运行”，而不是单纯的聊天机器人。

主视觉要求：
- 主视觉为一只“机械化微型龙虾（MicroClaw）”的黑白线描插画。
- 造型应精密、克制、略带科学图谱气质，而不是卡通、萌化或怪兽化。
- 细节可融入电路、齿轮、铰接结构、金属节理、微型工具臂等元素。
- 插画应保持单主体，避免复杂背景抢夺标题层级。

版式要求：
- 2:3 竖版。
- 顶部保留窄信息栏或眉题区域，但不要出现任何真实出版社品牌。
- 中上部放置书名，必须清晰、可读、层级最高。
- 副标题放在书名下方，字号明显小一档。
- 中央区域为主视觉插画。
- 底部可用一条克制的色带或信息区压住版面。
- 留白必须充足，不要把元素塞满。

配色要求：
- 背景为浅米色、暖灰纸张或微弱纸纹理，不要纯白，不要深色大底。
- 主体颜色以墨黑、炭灰、深灰为主。
- 强调色仅允许少量使用，建议锈橙、铜绿、旧金色三者择一。
- 整体应当偏印刷品气质，而不是数码 UI 气质。

排版与字体气质：
- 字体感觉应当稳重、清晰、出版级，不要花哨装饰。
- 可加小号英文眉题：MICROCLAW，但只能作为辅助信息。
- 中文标题要完整、准确、无错字、无变体字、无乱码。

风格关键词：
technical book cover, editorial publishing, engraved scientific illustration, monochrome ink drawing, precise linework, restrained composition, print-ready, engineering systems, archival paper texture, modern technical typography

## 反向约束

禁止项：
- 禁止任何 O'Reilly 字样、动物版式复刻、logo、商标或高度相似布局。
- 禁止霓虹赛博风、重金属游戏风、过度发光、悬浮 UI 面板、全息 HUD。
- 禁止廉价 AI 拼贴感、错误透视、肢体结构混乱、细节糊成一团。
- 禁止把龙虾画成食物、海鲜广告、卡通吉祥物或儿童插图。
- 禁止文字错字、乱码、低质量字体、排版拥挤。

## 输出要求

- 输出高分辨率封面图，适合 PDF 封面使用。
- 优先给出 3 到 4 个方向不同但都可出版的版本，再挑一个继续精修。
- 精修版本应重点优化：标题可读性、主视觉细节、留白、整体层级。

## 参数建议

- 比例：1024x1536 或更高
- 质量：high
- 输出数量：先 4 张，再精修 1 张
- 若支持 negative prompt，请将“品牌复刻、霓虹赛博、卡通龙虾、乱码标题、拥挤排版、低质量字体”加入负向约束
