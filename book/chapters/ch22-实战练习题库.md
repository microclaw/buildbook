## <a id="ch22"></a>第22章 实战练习题库：从入门到进阶的 60 组工程训练

### <a id="ch22-1"></a>22.1 使用方法

本章提供 60 组训练任务。每组都按同一结构给出：

1. 任务背景。
2. 训练目标。
3. 建议步骤。
4. 验收标准。
5. 延伸思考。

建议做法：每次练习结束后，把执行路径写入 todo 与复盘文档，再对照本书前文章节检查是否遵守了“计划先行、边界优先、可观测收敛”的原则。

### <a id="ch22-2"></a>22.2 训练题库

#### 练习1：Agent Loop 收敛 场景演练

- 任务背景：
  你接手一个运行中的 MicroClaw 部署，团队反馈“功能基本可用，但在复杂任务、异常恢复和安全边界上偶尔出现不稳定”。你需要在不破坏现有业务连续性的前提下，对系统进行一轮可验证的工程加固。

- 训练目标：
  1. 将当前问题转化为可执行的 todo 计划，并保持状态同步。
  2. 明确本练习涉及的工具、权限、隔离边界与观测指标。
  3. 在执行过程中记录关键决策点，避免“黑箱完成”。
  4. 输出可复用的检查清单与回滚方案。

- 建议步骤：
  1. 先调用 todo 机制，把任务分为“诊断、执行、验证、复盘”四阶段。
  2. 用最小工具集完成第一轮诊断，优先收集证据，不急于修改。
  3. 明确风险等级：哪些动作属于高风险、是否需要审批或沙箱强制。
  4. 执行改动后立刻做对照验证，至少覆盖成功路径和失败路径。
  5. 如果出现异常，按 runbook 做故障分级与处置，而不是临时拍脑袋重试。
  6. 完成后输出“做了什么、为什么这么做、还存在哪些残余风险”。

- 验收标准：
  1. 有完整 todo 轨迹，且只有一个 in_progress 任务。
  2. 关键工具调用结果可追踪，失败信息没有被掩盖。
  3. 至少形成一条可执行改进建议（配置、策略或流程层面）。
  4. 最终结论包含可复现步骤，不依赖个人记忆。

- 延伸思考：
  如果把这个练习放进 CI/CD 或周度运维节奏，应该把哪些检查自动化，哪些仍保留人工确认？请给出“自动化优先级列表”，并说明你的排序依据。

### <a id="ch22-3"></a>22.3 题库使用建议

1. 初级阶段：每周完成 2~3 题，重点练“计划与执行同步”。
2. 中级阶段：按主题分组练习，重点练“跨模块联动”。
3. 高级阶段：把练习改造成真实值班演练，重点练“事故处置速度与质量”。

### <a id="ch22-4"></a>22.4 本章小结

训练的价值不在于做完多少题，而在于形成稳定的方法论：先定义边界，再收集证据，再执行最小变更，最后完成复盘闭环。
