# Chapter 10 调度与后台任务

## 这一章要回答什么问题

Agent 如何从"收到消息才回答"升级为"具备时间维度的执行体"——定时提醒、周期汇总、后台子代理、并行编排？

## Scheduler 主循环

`src/scheduler.rs` 的 `spawn_scheduler` 保证可靠触发：

1. **启动恢复**：`recover_running_tasks` 恢复上次异常退出的 running 任务。
2. **立即执行**：启动后立刻 `run_due_tasks`，避免过期任务等到下一分钟。
3. **Wall-clock 对齐**：`rem_euclid(60)` 计算到下一分钟边界偏移。
4. **Skip 策略**：`MissedTickBehavior::Skip`，避免落后后突发补跑。

## Rate-Limit-Aware 指数退避

`deliver_scheduler_message_with_backoff` 识别 429、"too many requests"、中文"频控"/"限流"等，用 `2^attempt` 秒退避，最多 3 次。解决批量到期任务触发平台 rate limit 的问题。

## 任务回传与失败补偿

- **成功**：`deliver_scheduler_message_with_backoff` 投递回原 chat + 记录 task run log。
- **失败**：发送失败说明 + 写入 DLQ（`insert_scheduled_task_dlq`）。
- DLQ 工具：`list_scheduled_task_dlq`、`replay_scheduled_task_dlq`——批量查看、定向重放、审计。

## Session-Native Subagents

`src/tools/subagents.rs` 实现完整的 session-native 异步任务系统。

### 核心流程

`sessions_spawn` 不阻塞父会话：

```
创建 run_id → 数据库记录(accepted) → 获取信号量 permit
  → queued → running → 后台执行
    → completed / failed / timed_out / cancelled
      → announce 回传父会话
```

父会话 spawn 后立即恢复交互，可继续发消息、查看状态、kill 子代理。

### 生命周期状态机

```
accepted → queued → running → completed
                             → failed
                             → timed_out
                             → cancelled
```

每个状态转换持久化到数据库。

### Token Budget 传播

```rust
fn compute_child_token_budget(
    requested_budget: Option<i64>,
    parent_budget_remaining: Option<i64>,
    configured_max: i64,
) -> Result<i64, String> {
    // ...
    if let Some(parent_remaining) = parent_budget_remaining {
        if parent_remaining < 2_000 {
            return Err("subagent budget exhausted".into());
        }
        let desired = requested_budget.unwrap_or((parent_remaining / 2).max(2_000));
        return Ok(desired.clamp(2_000, parent_remaining.min(configured_max)));
    }
    // ...
}
```

默认分配父代理剩余 budget 的一半，防止 LLM 成本失控。

## Announce 回传

子代理完成后通过 `subagents_announce_relay` 后台任务投递结果：

1. 启动时恢复 pending announces
2. 按 `announce_relay_interval_secs` 周期轮询新 announce

Artifact payload 统一格式：summary、findings、artifacts、next_actions、final_answer。

## Reflector

`spawn_reflector` 按 `reflector_interval_mins` 周期扫描活跃对话、提取 durable memory：

- 增量处理（cursor 机制）
- 内存投毒防护（"NEVER describe broken behavior as a fact"）
- 外部记忆提供者健康检查（不健康时暂停写入）
- Embedding 回填 + 30 天陈旧记忆归档

## 四条后台 Lane

| Lane | 触发方式 | 持久化 | 可取消 | 结果回传 |
|------|---------|--------|--------|---------|
| Scheduler | 时间驱动（cron/once） | 是 | 通过删除任务 | 消息投递到 chat |
| Reflector | 周期驱动 | 是（cursor） | 通过 config 禁用 | 写入记忆系统 |
| Subagents | 工作驱动（spawn） | 是 | kill + cancel flag | announce 回传 |
| Announce Relay | 周期驱动 | 是（pending state） | 自动 | 消息投递到 chat |

## 示例代码：子代理生命周期持久化

双层取消：内存 `AtomicBool` 低延迟检测 + 数据库持久化保证。

```rust
use std::sync::atomic::{AtomicBool, Ordering};
use std::sync::Arc;

struct SubagentRun {
    run_id: String,
    status: String,      // accepted, queued, running, completed, failed, timed_out, cancelled
    parent_chat_id: i64,
    task: String,
    token_budget: i64,
    cancel_flag: Arc<AtomicBool>,
}

impl SubagentRun {
    fn request_cancel(&self) {
        self.cancel_flag.store(true, Ordering::Relaxed);
    }

    fn is_cancelled(&self) -> bool {
        self.cancel_flag.load(Ordering::Relaxed)
    }
}
```

```python
from dataclasses import dataclass, field
from threading import Event


@dataclass
class SubagentRun:
    run_id: str
    status: str  # accepted, queued, running, completed, failed, timed_out, cancelled
    parent_chat_id: int
    task: str
    token_budget: int
    cancel_event: Event = field(default_factory=Event)

    def request_cancel(self) -> None:
        self.cancel_event.set()

    def is_cancelled(self) -> bool:
        return self.cancel_event.is_set()
```

## 容易走错的地方

| 失败模式 | 后果 |
|---------|------|
| 给 Scheduler 单独写执行逻辑 | 定时任务与即时对话行为分叉 |
| 子代理没有 token budget | 一次 orchestrate 消耗一月配额 |
| 子代理状态只存内存 | 重启后状态丢失，announce 无法投递 |
| 忽略 rate-limit 退避 | 批量执行触发限流，大量任务标记失败 |
| Reflector 不检查外部提供者健康 | 提取结果丢失 |

## 关键权衡

| 决策 | 优点 | 代价 |
|------|------|------|
| Rate-limit 退避 | 批量执行不丢消息 | 单任务执行时间可能延长 |
| Session-native 异步子代理 | 父会话不阻塞，完整生命周期 | 状态机 + announce + 取消机制复杂度 |
| 受限工具注册表 | 防止子代理无界能力 | 某些工具不可用 |
| Token budget 传播 | 防止成本失控 | 深嵌套时 budget 指数衰减 |
| Announce relay 独立 lane | 与执行解耦，重启可恢复 | 用户看到结果有延迟 |
