# Chapter 5 项目骨架与工程基线

## 条目列表
- 规划仓库结构与职责拆分（core/storage/tools/channels/app）。
- 明确入口链路：CLI 命令 -> Runtime 组装 -> 服务启动。
- 定义统一错误模型、日志规范、配置加载机制。
- 列出工程基线：lint、测试、构建、发布前检查。

- 配置系统

## 写作思路
- 先讲“仓库为什么这样拆”，再讲每层如何协作。
- 用“新增功能落点”示例验证骨架是否可持续演进。
- 把工程基线写成执行清单（构建/测试/发布前检查）。
