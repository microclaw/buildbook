# Chapter 8 记忆系统

## Agent 怎样才能不每次都从零开始？

MicroClaw 的记忆系统是双层架构：文件记忆（`AGENTS.md`）解决可读与可编辑，结构化记忆（SQLite + 可选 `sqlite-vec`）解决检索与生命周期。`MemoryBackend` 支持本地 SQLite 或 MCP 外接，Reflector 负责自动积累，`memory_quality` 提供质量门控。

## 第一层：`AGENTS.md` 文件记忆

`MemoryManager` 通过 `build_memory_context` 按作用域（全局、bot/account、chat）装载文件记忆。

适合放高层规则和稳定背景（团队约定、工作目录、风格要求）。对人类友好、天然可审计。但难以做细粒度检索，不适合大量短事实。

## 第二层：结构化记忆与 `MemoryBackend`

`Memory` 结构含完整元数据：`id`、`chat_id`、`content`、`category`（PROFILE/KNOWLEDGE/EVENT）、`confidence`（0.0-1.0）、`source`、`last_seen_at`、`is_archived`、`embedding_model`。

`MemoryBackend` 默认 `local_only`（SQLite），也支持 MCP 外接。核心接口：`get_memories_for_context`、`insert_memory_with_metadata`、`supersede_memory`。上层代码（Reflector、显式记忆、检索注入）无需关心后端选择。

## 为什么必须双层

| 维度 | 文件记忆（AGENTS.md） | 结构化记忆（SQLite） |
| --- | --- | --- |
| 适合内容 | 声明性规则、稳定背景 | 事实性增量、对话积累 |
| 维护方式 | 人类直接编辑 | Agent/Reflector 自动管理 |
| 检索方式 | 按作用域全量装载 | 关键词/KNN 相关性排序 |
| 生命周期 | 手动维护 | 置信度、归档、supersede |

## 显式记忆 fast-path

检测"记住 X"/"remember X"后直接处理，绕过完整 agent loop：

```
内容提取 → 质量检查 → Jaccard 去重（阈值 0.55）
  → Topic 冲突走 supersede → 写入（confidence=0.95, source="explicit"）
    → 生成 embedding（如启用）
```

对显式意图，结构化逻辑比让模型自由发挥更可靠。

## 检索注入

`build_db_memory_context` 按三条路径检索：

```
sqlite-vec + embedding 可用？ ──是──→ KNN 语义检索
       │
       否 → 关键词打分（支持 CJK bigram）
```

**Token 预算**：`memory_token_budget`（默认 1500），按每 4 字符 ≈ 1 token 逐条累加，超预算停止。

注入格式：
```xml
<structured_memories>
[KNOWLEDGE] [chat] User prefers concise responses
[PROFILE] [global] Team uses TypeScript for frontend
(+3 memories omitted)
</structured_memories>
```

**注入审计**：每次记录 `retrieval_method`、`candidate_count`、`selected_count`、`omitted`、`used_tokens`——没有审计就无法调优检索策略。

## `sqlite-vec`：可选增强层

启用时：启动创建向量索引 → 优先 KNN 检索 → 写入时自动 embedding → `reembed` 命令批量重建。未启用时退化到关键词打分，系统完全正常工作。先把结构化治理做好，再加语义增强。

## Reflector：自动提取记忆

后台周期运行（默认 15 分钟），读取最近对话调 LLM 提取结构化记忆。`apply_reflector_extractions` 执行完整质量门控：

```
JSON 解析 → 类别校验 → 内容标准化（截断 180 字符）→ 毒性检测
  → 质量检查 → Supersede 处理 → Topic 冲突 → 去重（Jaccard ≥ 0.85）→ 合并决策
```

**毒性检测**：`should_skip_memory_poisoning_risk` 过滤"把错误行为描述成事实"的记忆（如 "tool calls were broken"），但保留纠正性行动项（以 "TODO:" 开头或含 "ensure"）。一旦错误行为被当事实长期注入，Agent 会不断强化偏差。

## 生命周期管理

- **更新**：`update_memory_with_metadata`
- **Supersede**：归档旧记忆 + 创建新记忆，保留关联
- **归档**：`is_archived=true`
- **删除**：`structured_memory_delete` 工具
- **最近使用**：`last_seen_at` 在注入时更新

Agent 通过 `structured_memory_search/update/delete` 主动管理记忆。

## 示例代码：显式 remember 受控写入

```rust
#[async_trait::async_trait]
trait MemoryStore {
    async fn insert_memory(
        &self, chat_id: i64, content: &str, category: &str,
        confidence: f32, source: &str,
    ) -> anyhow::Result<i64>;
    async fn update_memory(&self, memory_id: i64, content: &str) -> anyhow::Result<()>;
    async fn find_similar(&self, chat_id: i64, content: &str, threshold: f64)
        -> anyhow::Result<Option<i64>>;
}

struct ExplicitMemoryWriter<S> {
    store: S,
}
```

```rust

impl<S: MemoryStore> ExplicitMemoryWriter<S> {
    async fn remember(&self, chat_id: i64, content: &str) -> anyhow::Result<String> {
        if !memory_quality_ok(content) {
            return Ok("Skipped: content too vague.".into());
        }
        if let Some(existing_id) = self.store.find_similar(chat_id, content, 0.55).await? {
            self.store.update_memory(existing_id, content).await?;
            return Ok(format!("Updated memory #{existing_id}"));
        }
        let id = self.store.insert_memory(chat_id, content, "KNOWLEDGE", 0.95, "explicit").await?;
        Ok(format!("Saved memory #{id}"))
    }
}
```

## 关键权衡

| 决策 | 优点 | 代价 |
| --- | --- | --- |
| 文件+结构化双层 | 兼顾可编辑和可检索 | 需解释职责边界 |
| `MemoryBackend` 支持 MCP | 可演进到外部服务 | 需保持行为一致 |
| 显式记忆 fast-path | 可靠、可预测、省 LLM 成本 | 额外维护一条非通用路径 |
| Reflector 默认开启+多层门控 | 自动积累事实 | 门控逻辑需持续维护 |
| 向量检索为增强层非强依赖 | 启动简单、默认稳定 | 纯本地语义召回有限 |

## 容易走错的地方

1. **所有信息塞 `AGENTS.md`**：文件记忆适合少量高层规则，不适合大量细粒度事实。
2. **结构化记忆当无上限日志**：没有置信度、归档和 supersede，旧事实不断稀释新事实。
3. **自动提取忽视污染风险**：错误行为写成长期事实 → Agent 自我强化偏差。
4. **没有注入审计就调优检索**：不知道注入了多少、截断了多少，无法判断记忆是否帮助了模型。
5. **过早依赖 embedding**：Reflector 毒性检测没做、归档策略没定、审计不存在 → 语义检索召回的全是低质量记忆。

## 小结

MicroClaw 的记忆系统把记忆做成可编辑、可检索、可观测、可归档、可纠错的双层系统。`sqlite-vec` 作为可选增强而非强依赖，毒性检测和质量门控确保自动提取不成为污染源。
