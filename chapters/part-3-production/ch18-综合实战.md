# Chapter 18 综合实战

## 落地路线

MicroClaw 提供完整运维工具链：Setup wizard（全屏 TUI）、Doctor 诊断、Gateway 服务管理、Upgrade 自升级、Reembed 向量重建。按三阶段落地：

- **Day 1**：setup + doctor + start → 最小可运行
- **Week 1**：渠道、记忆、并行工具执行
- **Month 1**：安全、观测、门禁、升级全部就绪

## Day 1：最小可运行版本

### 安装

```bash
brew install microclaw/tap/microclaw          # Homebrew
curl -sSf .../install.sh | bash               # One-line
docker compose up -d                           # Docker
nix run github:microclaw/microclaw             # Nix
cargo install --path . --features full         # 源码
```

### Setup Wizard

```bash
microclaw setup
```

引导完成：LLM Provider 配置（验证连通性）→ 渠道配置 → Sandbox 模式 → 高级选项（SOUL.md、MCP、记忆）→ 生成 `microclaw.yaml`。

### Doctor 诊断

```bash
microclaw doctor          # 结构化诊断
microclaw doctor --json   # 自动化友好
```

覆盖：平台信息、配置文件、LLM Provider、Sandbox、渠道、MCP、数据库、Web UI。每项状态：`PASS` / `MISS` / `WARN` / `FAIL`。

### 启动与验收

```bash
microclaw start
```

Day 1 验收四项：基础对话成功、多轮上下文保持、工具调用返回、Web 控制面可见。

### 设置 Web 密码

```bash
microclaw web password "your-strong-password"
microclaw web password-generate              # 随机密码
microclaw web password-clear                 # 清除（测试用）
```

### Day 1 最小 smoke check

```rust
struct SmokeCheck {
    can_chat: bool,
    can_store_session: bool,
    can_read_metrics: bool,
    web_password_set: bool,
}

impl SmokeCheck {
    fn passed(&self) -> bool {
        self.can_chat && self.can_store_session
            && self.can_read_metrics && self.web_password_set
    }
}
```

```python
from dataclasses import dataclass


@dataclass
class SmokeCheck:
    can_chat: bool
    can_store_session: bool
    can_read_metrics: bool
    web_password_set: bool

    def passed(self) -> bool:
        return all([
            self.can_chat, self.can_store_session,
            self.can_read_metrics, self.web_password_set,
        ])
```

## 示例代码：最小 Runtime 骨架

```rust
#[async_trait::async_trait]
trait LlmProvider {
    async fn send(&self, prompt: &str) -> anyhow::Result<String>;
}

struct AppState {
    llm: Box<dyn LlmProvider>,
    tools: ToolRegistry,
    turn_queue: Arc<ChatTurnQueue>,
    trace_exporter: Option<Arc<OtlpTraceExporter>>,
}

impl AppState {
    async fn run_once(
        &self, channel: &str, chat_id: i64, prompt: &str,
    ) -> anyhow::Result<String> {
        let _guard = self.turn_queue.acquire(channel, chat_id).await;
        let _tools = self.tools.list();
        self.llm.send(prompt).await
    }
}
```

```python
import asyncio
from dataclasses import dataclass
from typing import Protocol


class LlmProvider(Protocol):
    async def send(self, prompt: str) -> str: ...

@dataclass
class AppState:
    llm: LlmProvider
    tools: "ToolRegistry"
    turn_queue: "ChatTurnQueue"

    async def run_once(
        self, channel: str, chat_id: int, prompt: str,
    ) -> str:
        async with self.turn_queue.acquire(channel, chat_id):
            _tools = self.tools.list()
            return await self.llm.send(prompt)
```

## 容易走错的地方

### 失败模式 1：跳过 setup 和 doctor

Setup wizard 处理了大量边界情况。手写配置容易遗漏关键字段。

### 失败模式 2：Week 1 不启用并行工具执行

延迟可降 60%+，只需配置。

### 失败模式 3：Month 1 仍用默认密码

系统越有用，风险越高。

### 失败模式 4：不建升级回滚流程

没有备份和验证，升级就是赌博。
