# Chapter 18 综合实战

## 落地路线

MicroClaw v0.1.57 提供完整运维工具链：Setup wizard（全屏 TUI）、Doctor 诊断、Gateway 服务管理、Upgrade 自升级、Reembed 向量重建。把「能跑起来」「能稳定运行」「能托管交付」拆成三个阶段：

- **Day 1**：`microclaw setup --enable-sandbox` + `doctor` + `start` → 最小可运行
- **Week 1**：OTLP 导出、Hook 拦截、自定义 Skill、第一个 scheduled_task
- **Month 1**：Web 控制面 + API key + RBAC、MCP 工具联邦、sqlite-vec 记忆、A2A 节点联通

## Day 1：最小可运行

### 安装

```bash
brew install microclaw/tap/microclaw          # Homebrew
curl -sSf https://.../install.sh | bash       # one-liner
docker compose up -d                          # 容器
nix run github:microclaw/microclaw            # Nix
cargo install --path . --features full        # 源码
```

### 交互向导

```bash
microclaw setup --enable-sandbox
```

`setup.rs` 走完六步：

1. **LLM Provider**：从内置 preset（Claude、OpenAI、DeepSeek、Qwen 等）选一项，填 API key，向导会真的发一次握手验证连通性。
2. **渠道**：从 15 个内置渠道（telegram、discord、slack、feishu、weixin、qq、imessage、whatsapp、signal、matrix、irc、nostr、dingtalk、email 等，不同平台数量略有差异）里挑 1–2 个启用。
3. **Sandbox 模式**：`Off` / `Shared` / `All`，多用户场景至少 `Shared`。
4. **高级选项**：SOUL.md 启用、记忆持久化、MCP 列表、Hook 默认目录。
5. **写文件**：生成 `microclaw.config.yaml`，自动创建 `<data_dir>` 与 `hooks/`、`skills/`。
6. **首启**：可选直接启动 `microclaw start`。

### Doctor 诊断

```bash
microclaw doctor          # 终端结构化输出
microclaw doctor --json   # CI/自动化
```

覆盖：平台信息、配置文件、LLM Provider 连通性、Sandbox（`SandboxMode` + runtime + image + 挂载白名单）、渠道、MCP 依赖、数据库 schema、Web UI、`web_fetch` 校验是否 strict。状态：`PASS` / `MISS` / `WARN` / `FAIL`。

### 启动

```bash
microclaw start
```

Day 1 验收四项：

```rust
struct SmokeCheck {
    can_chat: bool,
    can_store_session: bool,
    can_read_metrics: bool,
    web_password_set: bool,
}

impl SmokeCheck {
    fn passed(&self) -> bool {
        self.can_chat
            && self.can_store_session
            && self.can_read_metrics
            && self.web_password_set
    }
}
```

## Week 1：从 Demo 到持续可用

| 目标 | 要点 |
|------|------|
| 启用 OTLP 导出 | 配置 `metric_exporter` / `trace_exporter` / `log_exporter` 端点，验证 dashboard 能看到 wave 级 span |
| 写自定义 skill | 在 `skills/` 下加 `SKILL.md` + 资源文件；`skill_activation_logs` 自动记录使用 |
| 配置 Hook 拦截高危工具 | `hooks/` 下放可执行脚本，`BeforeToolCall` 上对 `bash` 命令字符串做关键字检查 |
| 第一个 scheduled_task | 先建低风险 once 任务（每天发个总结），观察调度命中率 |
| 启用并行工具执行 | 默认即开，验证 ToolWave 事件、ReadOnly 工具确实并行、延迟下降 |
| 接入 1–2 个 MCP server | 提前暴露扩展治理问题；配置 `circuit_breaker` 与 `rate_limit_per_minute` |
| Gateway 服务化 | `microclaw gateway install && microclaw gateway start`（按平台生成 systemd / launchd / Windows Service） |

## Month 1：可托管状态

### 安全加固

- 密码或 bootstrap token 已设置，未使用默认值
- `ToolRisk::High` 工具确认机制保持开启
- Sandbox 至少 `Shared`，多用户必须 `All`
- Web API key 全部 scoped（`operator.read` / `operator.write` / `operator.admin` / `operator.approvals`）
- `web_fetch.block_private_ips=true`，URL 与 content 双校验启用
- `audit_logs` 持续增长，每周抽查异常 actor

### 控制面与 RBAC

```bash
# 创建一个仅有读权限的 API key
curl -X POST /api/auth/api_keys \
  -H "Cookie: mc_session=..." \
  -d '{"label":"dashboard-readonly","scopes":["operator.read"]}'
```

返回值里的 `api_key` 字段是唯一一次明文展示，落库只存 SHA-256；管理界面只显示 prefix。

### MCP 工具联邦

```yaml
mcp_servers:
  - name: notion
    command: npx
    args: ["-y", "@modelcontextprotocol/server-notion"]
    request_timeout_secs: 30
    rate_limit_per_minute: 30
    circuit_breaker_failure_threshold: 5
  - name: github
    command: npx
    args: ["-y", "@modelcontextprotocol/server-github"]
```

每个 server 都跑在独立子进程，失败被限流和熔断隔离。

### sqlite-vec 与记忆

```bash
cargo install --path . --features sqlite-vec
microclaw reembed                  # 全量重嵌入
```

启用后 `microclaw_memory_search` 走向量召回；`memory_token_budget` 限制注入到 LLM 上下文的总量。

### A2A 节点联通

```yaml
a2a:
  enabled: true
  listen: "0.0.0.0:7301"
  peers:
    - name: browser-rig
      url: "https://browser-rig.example.com"
```

主实例通过工具 `a2a_send` 把任务委派给 `browser-rig`，远端实例的 `agent-card` 在 `/api/a2a/agent-card` 自描述能力。

### 升级与回滚

```bash
cp <data_dir>/microclaw.db <data_dir>/microclaw.db.bak-$(date +%F)
microclaw upgrade
microclaw gateway restart
microclaw doctor                   # 升级后再跑一遍
```

升级失败回滚：把上一份二进制换回去，把备份 db 恢复，`gateway restart`。

### 子代理预算审计

```bash
sqlite3 <data_dir>/microclaw.db <<SQL
SELECT status, COUNT(*), SUM(token_usage), printf("%.2f", SUM(cost_estimate))
FROM subagent_runs
WHERE started_at > datetime('now','-7 day')
GROUP BY status;
SQL
```

把这条查询挂进每周 runbook，预算异常会立刻显形。

## 最小可交付 MicroClaw 检查清单

```
[配置]
  □ microclaw.config.yaml 存在且通过 microclaw doctor
  □ Web 密码已设置（非默认）或 bootstrap token 已使用一次
  □ ToolRisk::High 确认开启，web_fetch 校验启用且 strict
  □ Sandbox 至少 Shared

[健康端点]
  □ /api/health 返回 200
  □ /api/metrics 与 /api/metrics/summary 可访问
  □ /api/a2a/agent-card 自描述完整（如启用 A2A）

[数据]
  □ <data_dir>/microclaw.db 每日备份
  □ schema_version 与代码版本一致（v25+）
  □ subagent_runs / metrics_history / audit_logs 有持续写入

[可观测]
  □ OTLP 端点可达或本地 metrics_history 持续增长
  □ Langfuse / AgentOps 看板能看到 wave 级 span（若接入）

[运维]
  □ microclaw gateway status 正常
  □ 升级流程跑过一次（含回滚演练）
  □ Runbook 包含：告警 → 看板 → 表 → 回滚四个步骤
```

## 最小 Runtime 骨架

```rust
use std::sync::Arc;

#[async_trait::async_trait]
pub trait LlmProvider: Send + Sync {
    async fn send(&self, prompt: &str) -> anyhow::Result<String>;
}

pub struct AppState {
    pub llm: Box<dyn LlmProvider>,
    pub tools: ToolRegistry,
    pub turn_queue: Arc<ChatTurnQueue>,
    pub trace_exporter: Option<Arc<OtlpTraceExporter>>,
}

impl AppState {
    pub async fn run_once(
        &self,
        channel: &str,
        chat_id: i64,
        prompt: &str,
    ) -> anyhow::Result<String> {
        let _guard = match self.turn_queue.acquire(channel, chat_id).await {
            Some(g) => g,
            None => return Ok("busy".into()),
        };
        let _tools = self.tools.list();
        let answer = self.llm.send(prompt).await?;
        if let Some(exp) = &self.trace_exporter {
            exp.record_run(channel, chat_id, prompt.len(), answer.len());
        }
        Ok(answer)
    }
}
```

## 容易走错的地方

**跳过 setup 和 doctor**：手写配置容易遗漏关键字段，`doctor` 是部署前的最后一道关。

**Week 1 不启用 OTLP 与 Hook**：等出事故再回头加观测和拦截，意味着第一次事故没有上下文可看。

**Month 1 还在默认密码**：系统越有用，控制面越值钱，默认密码就是把钥匙挂门外。

**没有备份与回滚演练**：升级是赌博。把「备份 db → 升级 → doctor → 回滚演练」的流程写进 runbook 才算闭环。

**MCP 不配熔断**：上游一抖，整个 runtime 跟着卡死。
